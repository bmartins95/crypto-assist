Vínculo de operações — tag + resumo flutuante
Uma tag de ciclo (ex.: A3) marca todas as operações que pertencem ao mesmo par entrada↔saída. Ao passar o mouse sobre qualquer tag — a da entrada ou a de uma saída — abre um resumo flutuante do ciclo inteiro, incluindo saídas parciais e o restante em aberto. A tabela não é reordenada; os grupos por dia permanecem intactos.

1. Modelo de dados
Cada operação ganha um campo opcional cycleId. Todas as operações do mesmo par entrada↔saída compartilham o mesmo cycleId. Uma posição solta (sem fechamento) tem cycleId = null e não exibe tag.
type Operation = {
  id: string
  cycleId: string | null   // ex.: "A3" — null = posição solta
  role: 'entry' | 'exit'   // entrada (compra) ou saída (venda)
  coin: string             // "USDT"
  date: string             // ISO — mantém o agrupamento por dia
  qty: number              // quantidade desta operação
  total: number            // valor em BRL desta operação
  pnl: number | null       // realizado nesta operação (null p/ entrada)
  status: 'aberta' | 'parcial' | 'fechada'
}

// Um ciclo é derivado, não armazenado:
type Cycle = {
  cycleId: string
  entry: Operation             // 1 entrada
  exits: Operation[]           // 1..N saídas (parciais)
  qtyEntry: number
  qtyClosed: number            // Σ exits.qty
  qtyRemaining: number         // qtyEntry - qtyClosed
  realizedPnl: number          // Σ exits.pnl
  status: 'parcial' | 'fechada'  // fechada quando qtyRemaining ≈ 0
}
Agrupe as operações por cycleId no cliente (um Map<cycleId, Cycle>) uma vez por render da lista. A tabela continua ordenada e agrupada por date — o ciclo é apenas uma visão sobreposta.
2. A tag na linha
  • Renderize a tag apenas quando op.cycleId != null, logo após o chip de tipo (Compra/Venda).
  • Rótulo = cycleId. Mesma tag idêntica na entrada e em cada saída — é o que comunica o vínculo mesmo em dias diferentes.
  • Ícone de elo (link) + label. Cor roxa (#c4b5fd) para não competir com verde/vermelho de compra/venda.
  • cursor: pointer. A tag é o único gatilho do resumo — o resto da linha mantém o comportamento atual (expandir detalhes / ações).
3. O resumo flutuante
Popover posicionado relativo à tag (position:absolute dentro de um wrapper position:relative). Conteúdo idêntico independentemente de qual tag disparou — ele descreve o ciclo, não a operação.
Estrutura, de cima para baixo:
  1. Cabeçalho — "Ciclo {coin} · {cycleId}" + badge de status (Parcial se qtyRemaining > 0, senão Encerrado).
  2. Linha de entrada — seta ↓ verde, "Compra · {data}", qtd e total.
  3. Uma linha por saída — seta ↑ vermelha, "Venda parcial · {data}" (ou "Venda ·" se for única e total), qtd e total. N saídas = N linhas.
  4. Restante em aberto — só aparece quando qtyRemaining > 0. Ícone neutro, quantidade restante, total "—".
  5. Rodapé — "Resultado realizado" = realizedPnl (soma das saídas). Verde/vermelho conforme sinal. Rotule como realizado, não "líquido", enquanto houver restante.
4. Interação e estados
  • Hover abre; sair fecha. Transição de ~160ms em opacity + translateY(-6px→0).
  • Guarde só o id da operação em hover (ou null) no estado. A tag da entrada e a da saída são gatilhos independentes; ambas resolvem para o mesmo cycleId e mostram o mesmo popover.
  • z-index alto e overflow:visible no contêiner da tabela para o popover não ser cortado.
  • Flip vertical: se a linha estiver na metade de baixo da viewport, abra o popover acima da tag (bottom: 100%) em vez de abaixo.
  • Mobile / toque: sem hover — a tag vira tap que abre o mesmo resumo como bottom-sheet/popover, com fechar por tap-fora.
  • Acessibilidade: tag é button com aria-label="Ver ciclo {cycleId}"; abrir também no focus, fechar no Esc.
5. Saídas parciais — regras
  • Uma entrada pode ter várias saídas com o mesmo cycleId. Cada saída é uma linha própria na tabela, no seu respectivo dia.
  • Status da entrada e das saídas = Parcial enquanto qtyRemaining > 0; todas viram Fechada quando a última saída zera o restante.
  • Cada linha de saída mostra no L/P apenas o realizado daquela saída. O acumulado do ciclo vive só no rodapé do popover.
  • Use tolerância (ex.: Math.abs(qtyRemaining) < 1e-8) para decidir "fechada" e evitar resíduo de ponto flutuante.

7. Aceite
  • Operações com cycleId mostram a tag; posições soltas não.
  • Hover em qualquer tag do mesmo ciclo abre o mesmo resumo, sem mover linhas nem alterar os grupos por dia.
  • Resumo lista entrada + todas as saídas + restante em aberto + resultado realizado.
  • Ciclo com restante > 0 aparece como Parcial; ao zerar, tudo vira Fechada.
  • Popover nunca é cortado pela borda da tabela nem pela borda da viewport (flip vertical).
  • Funciona com teclado e em toque (tap abre/fecha).
