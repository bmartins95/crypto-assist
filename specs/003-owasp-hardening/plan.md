## Phase 0: Research (Complete)

See [research.md](research.md) for all five technical decisions. Summary:

1. **Isolation tests**: Inspect `cur.execute.call_args` to verify the authenticated user's ID is used in queries; configure mock to return `[]` for user B. *(research.md Decision 1)*
2. **coin_id validation**: `re.fullmatch(r'^[a-z0-9-]{1,120}$', cid)` on each ID before cache query; reject entire request on any failure with HTTP 400. *(research.md Decision 2)*
3. **Auth failure logging**: Inject `Request` into `require_auth`; log `WARNING` with `request.url.path` and `request.headers.get("user-agent")`. Never log the token. *(research.md Decision 3)*
4. **CSP header**: `aws.cloudfront.ResponseHeadersPolicy` in `app-stack.ts` attached to `defaultCacheBehavior.responseHeadersPolicyId`. Interpolate backend URL from SSM. *(research.md Decision 4)*
5. **SHA pinning**: Retrieve SHAs via `git ls-remote` at implementation time; pin all three actions; add `# vN` comment on each line. *(research.md Decision 5)*

### User Story 3 — Auth failure logging (A09)

**File**: `backend/app/dependencies.py` — updated signature:

```python
import logging
from fastapi import Header, HTTPException, Request, status
from app.cognito import decode_token

logger = logging.getLogger(__name__)

def require_auth(
    request: Request,
    authorization: str | None = Header(default=None),
) -> AuthContext:
    if not authorization or not authorization.startswith("Bearer "):
        logger.warning("auth_failure path=%s ua=%s reason=missing",
                       request.url.path,
                       request.headers.get("user-agent", "-"))
        raise HTTPException(status_code=401, detail="Missing authentication token.")
    token = authorization.removeprefix("Bearer ")
    try:
        claims = decode_token(token)
    except Exception:
        logger.warning("auth_failure path=%s ua=%s reason=invalid",
                       request.url.path,
                       request.headers.get("user-agent", "-"))
        raise HTTPException(status_code=401, detail="Invalid or expired token.")
    return AuthContext(user_id=claims["sub"])
```

The 401 tests in `test_isolation.py` cover this logging path indirectly (every 401 request will produce a WARNING).

### User Story 4 — CSP header (A05)

**File**: `aws-infra/stacks/app-stack.ts` — inside the `if (config.web?.cloudFront)` block, before the distribution resource:

```typescript
const backendApiUrl = aws.ssm.getParameterOutput({
  name: `${paramBase}/BackendApiUrl`,
});

const cspPolicy = new aws.cloudfront.ResponseHeadersPolicy(`${prefix}WebCspPolicy`, {
  securityHeadersConfig: {
    contentSecurityPolicy: {
      contentSecurityPolicy: $util.interpolate`default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self' https://cognito-idp.us-east-1.amazonaws.com https://*.amazoncognito.com ${backendApiUrl.value}`,
      override: true,
    },
  },
});
```

Then in `defaultCacheBehavior`:
```typescript
defaultCacheBehavior: {
  // ... existing fields ...
  responseHeadersPolicyId: cspPolicy.id,
},
```

**Verification**: After deploy, `curl -I https://<cloudfront-domain>` must show `content-security-policy` header.
