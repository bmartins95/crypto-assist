Handoff — Per-asset charts & richer chart tooltips
For Claude Code — implementation spec for the Profit / Portfolio value area. Visual mocks live in Gráficos - Ideias por Ativo e Tooltips.dc.html; ids referenced below (2a, 2b, 1d, 1f) point to that file.
SCOPE — 4 pieces
  1. Component A (2a) — dual-axis companion on the existing wallet charts: overlay one asset's price over the wallet curve.
  2. Component B (2b) — a new, separate "Assets over time" component below the main chart: dense, sortable list of every asset with a sparkline.
  3. Profit tooltip — enrich to exactly 1d. Which-assets composition does NOT go in the tooltip; it surfaces in the asset rows of A and B, synced to the hovered day.
  4. Portfolio-value tooltip (1f) — current vs. invested + unrealized P/L.
1. COMPONENT A — Dual-axis companion (ref 2a)

Lives on the existing Profit over time and Portfolio value charts (same card, not a new tab). Adds a single-select "Compare with:" control that overlays one asset onto the current chart.

  • Selector is a radio (segmented): Nenhum · BTC · ETH · SOL · …. One asset at a time — never accumulates lines. Default Nenhum (chart unchanged from today).
  • Two independent Y axes. Left = wallet series in R$ (profit or value, as today). Right = the selected asset's % change over the period. Each axis scales to its own min/max so the asset curve never flattens the wallet curve. This is the fix for the scale problem.
  • Styling: wallet line keeps its current color/area (teal on value, periwinkle on profit). Asset overlay = dashed line in that asset's brand color. Legend shows both, with the asset entry labeled "(% no período)".
  • Right-axis ticks are tinted in the asset color so it's obvious which axis is which.

Persistence: remember the last compared asset per chart (nice-to-have, not required).

2. COMPONENT B — Assets over time list (ref 2b)

A separate component below the main chart. Replaces the "small multiples" idea — a card grid didn't scale past a handful of assets; this list does.

  • One row per asset: icon badge + name · sparkline (period) · price · period % change (green/red).
  • Header controls: search box + sort dropdown (biggest movement / alphabetical / allocation). Rows scroll inside a fixed-height container — handles 3 or 30+ assets identically.
  • Row click opens that asset's full chart (or drives Component A's selector to that asset — your call; both are fine).
  • Period follows the same 1D/1W/1M/1Y/All range control as the main chart.

Note: sparkline % is the last point of the cumulative %-change series (not delta of first/last — first is 0).

3. PROFIT TOOLTIP — enrich to exactly 1d

On hover over a day in the Profit chart, replace today's single-line popup with the 1d layout. Keep it to exactly these fields — do NOT list assets in the tooltip.

  • Date header (e.g. 23 jul 2026) + weekday.
  • Cumulative profit, large, colored green/red (e.g. +R$143.28).
  • Day delta: R$ vs. previous point + % (e.g. +R$58.90 no dia · +69.8%).
  • Divider, then rows: Realizado, Não realizado, Operações no dia.

Which assets composed the value is answered in the asset rows of Component A & B, synced to the hovered day: hovering a day highlights that day and the asset rows show that day's per-asset contribution. This keeps the tooltip lean (2c decision).

4. PORTFOLIO-VALUE TOOLTIP (ref 1f)

On the Portfolio-value chart, the hover popup shows both series plus the derived P/L:

  • Valor atual — teal solid swatch, R$.
  • Investido — periwinkle dashed swatch, R$ (muted text).
  • Highlighted block: Resultado não realizado = atual − investido, in R$ and %, colored green/red.
  • Variação no dia row at the bottom.
